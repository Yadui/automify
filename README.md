## Production Database Migrations

Vercel builds should not run production migrations automatically. The build script only runs `prisma generate` and `next build`; apply migrations intentionally with:

```bash
npm run db:migrate:deploy
```

If a production database was created before Prisma migration history was recorded, Prisma can fail with `P3018` and `relation "User" already exists`. After confirming the existing database schema matches `prisma/schema.prisma`, mark the baseline Lucia migration as applied once:

```bash
npm run db:migrate:resolve:lucia
npm run db:migrate:status
```

Run these commands with the production `DATABASE_URL` configured, and take a database backup before resolving migration history.
